# FeverCode Release Process

- No releases published yet

- dist-workspace.toml
  - Cargo-dist configuration is stored at the repository root in dist-workspace.toml. It defines the workspace, artifacts, and release behavior.

- Target platforms
  - x86_64-linux
  - aarch64-linux
  - x86_64-macOS
  - aarch64-macOS
  - x86_64-windows

- Triggering a release
  - Create a tag for the release, for example: git tag v0.1.0
  - Push the tag to the remote: git push origin v0.1.0

- Release workflow
  - The release workflow is generated by cargo-dist and placed at .github/workflows/release.yml. It builds artifacts for all target platforms when a tagged release is pushed.
- Artifact naming
  - Artifacts follow cargo-dist conventions and include the version and platform, e.g. fever-0.1.0-x86_64-linux.tar.gz.
