Install FeverCode

Choose your platform. One command to get started.

# One-line install (macOS / Linux)
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/FeverDream-dev/FeverCode/releases/latest/download/fever-installer.sh | sh
# One-line install (Windows PowerShell)
powershell -ExecutionPolicy Bypass -c "irm https://github.com/FeverDream-dev/FeverCode/releases/latest/download/fever-installer.ps1 | iex"
# Install from source (all platforms)
cargo install --git https://github.com/FeverDream-dev/FeverCode fever
One-line installers available after the first release. Use source install until then.

⚠ No release yet?

Until the first release is published, install from source:

git clone https://github.com/FeverDream-dev/FeverCode
cd FeverCode/fevercode_starter
cargo build --release

Safety Modes

Three operating modes. You choose how much autonomy FeverCode gets.

💬

ask

Ask Mode

Read-only. FeverCode analyzes your code, answers questions, suggests changes — but touches nothing. Safe as a conversation.

⚖

auto

Auto Mode

Proposes changes through the approval queue. Every edit, every file operation passes through your review before applying.

⚡

spray

Spray Mode

Full autonomy within the workspace boundary. FeverCode executes plans, writes code, runs tests — all without waiting for approval.

🛡

Workspace Boundary — Non-Negotiable

Regardless of mode, FeverCode never operates outside your project directory. No home directory access, no system files, no escapes. The workspace boundary is enforced at the architecture level — not a config flag you can toggle off.
